This week I did a really onerous work——I manually annotated the data which means we now have a test set! Me and Zhaoqing consider this is the least error-proof method for there are numbers that needed to be replaced by characters and punctuation needed to be deleted and other works needed to be done manually. I also think the deep understanding the dataset is important and it’s a opportunity to learn dataset in details. The ASR for news program like Xinwenlianbo is not like quotidian conversation. It has its own characteristcs. In this week, I also discuss with Qibin and Yifan who is the student of Chinese OCR. So, it’s quite a substantial week!

How to play audios in HPC

This is very important. Prof Steen installed Sox for us. Sox is a powerful tool for audio editting. However, for some reasons I didn’t know, I couldn’t use Sox in my image, but I can use it on my local computer. So, I guess it’s because of the cluster environment, maybe I don’t have the authorities.

Prof Steen advised me to use netcat.
Netcat is a featured networking utility which reads and writes data across network connections, using the TCP/IP protocol. I transfer all the audios to my computers and I found it really helpful for I don’t need to edit text in cluster. I can edit all the texts and transfer back to HPC. It’s less time-comsuming because there are often delays in cluster and sometimes my connection will automatically break in an hour.

How to use netcat to transfer the whole directory

Suppose that A is going to transfer files to B.

Firstly, B starts the listening

nc -l port | tar xfvz -

Notice that defining a new folder is not accepted. You might as well create a folder first.

Secondly, A starts packing all files under the current directory and sending to B

tar cfz - * |nc IPaddress Port

Use Chrome to play the audio

Qibin told us this method. It’s simple and convenient.

Firstly,start the server

python -m SimpleHTTPServer

Secondly，open Chrome and input hpclogin.case.edu:8000

And you can access to the directory and play the audio by Chrome.

One problem is that it needs uploading. A ten-second audio needs 10 seconds to upload and it’s definitely not a good experience.

If you want to check one or two audios, it’s best to use Chrome. But if you need to check all the audios like me, transfering to local is the best choice.

Annotations

Resplit or recombine some audios

There are several audio lasting for almost 1 minutes and I resplit them with shorter padding duration. Also, there are audios which last only 1 or 2 seconds and I combine them so they can make up a complete sentence.

Such problem is inevitable for everyone speaks at different speeds. After the testing, if we need to split all the audios, it’s necessary to filter out the long audios and split them again.

Some characteristics about Chinese news programs

As I said above, news program has its own characteristics.

  1. Wording and phrasing are quite succinct and brief. It’s even obscure for native speakers. For example “建言资政”, you can know the meaning by the character, but it’s hard to come out these characters immediately when listening.
  2. There are so many names——both person names and location names. And location names are usually not common, it might be a unknown village. If the pipeline can’t recognize the name and regard it as a word or even break the name and recombine them with neighbouring characters, the result won’t be satisfying.
  3. There are foreign languages, not much, but they exist in almost every episode.

The following work

Most news programs don’t have subtitles, but variety shows and TV plays have. Yifan said he will build the OCR pipeline in July, I guess that will match the time I begin to build NLP pipeline.
